# Портал "Корочки.есть" - Система записи на курсы дополнительного образования

## Описание проекта

Информационная система для записи на онлайн-курсы профессиональной переподготовки. Система позволяет пользователям регистрироваться, подавать заявки на обучение, а администраторам - управлять заявками и изменять их статусы.

## Технологический стек

- **Frontend**: HTML5, CSS3, JavaScript (Vanilla)
- **Backend**: PHP 7.4+
- **База данных**: MySQL/MariaDB
- **Архитектура**: RESTful API
- **Паттерны**: OOP, MVC

## Установка и настройка

### 1. Требования

- PHP 7.4 или выше
- MySQL 5.7+ или MariaDB 10.3+
- Apache/Nginx веб-сервер
- XAMPP/WAMP/LAMP (рекомендуется)

### 2. Настройка базы данных

1. Создайте базу данных:
```sql
mysql -u root -p < backend/database_schema.sql
```

Или импортируйте файл `backend/database_schema.sql` через phpMyAdmin.

2. Настройте подключение в `backend/config/database.php`:
```php
private $host = "localhost";
private $db_name = "korochnik";
private $username = "root";
private $password = "";
```

### 3. Настройка веб-сервера

Для XAMPP:
- Поместите папку `korochnik` в `htdocs/`
- Откройте в браузере: `http://localhost/korochnik/frontend/`

### 4. Настройка API

В файлах `frontend/app.js`, `frontend/dashboard.js`, `frontend/admin.js` проверьте:
```javascript
const API_BASE = 'http://localhost/korochnik/backend/api';
```

## Функциональность

### Модуль 1: Основная функциональность

#### Регистрация пользователей
- Уникальный логин (латиница и цифры, минимум 6 символов)
- Пароль (минимум 8 символов)
- ФИО (кириллица и пробелы)
- Телефон (формат: 8(XXX)XXX-XX-XX)
- Email (валидация формата)
- Все поля обязательны

#### Вход в систему
- Авторизация по логину и паролю
- Специальный вход для администратора (Admin / KorokNET)
- Навигация между формами входа и регистрации

#### Подача заявок
- Выбор курса из списка или ввод своего
- Указание желаемой даты начала обучения
- Выбор способа оплаты (наличными/переводом)
- Статус по умолчанию: "Новая"

#### Просмотр заявок
- Пользователи видят только свои заявки
- Администратор видит все заявки
- Отображение статуса, дат, способа оплаты

#### Административная панель
- Просмотр всех заявок
- Фильтрация по статусу
- Изменение статуса заявок:
  - Новая
  - Идет обучение
  - Обучение завершено
- Пагинация (6 заявок на страницу)
- Всплывающие уведомления

### Модуль 2: Дизайн и UX

#### Изображения и слайдер
- Слайдер с 4 изображениями на главной странице
- Автоматическая смена каждые 3 секунды
- Навигационные кнопки (вперед/назад)
- Индикаторы текущего слайда
- Пауза при наведении мыши

#### Валидация форм
- Валидация в реальном времени
- Отображение ошибок под полями
- Форматирование телефона автоматически

#### Курсы
Три обязательных курса в системе:
1. Основы алгоритмизации и программирования
2. Основы веб-дизайна
3. Основы проектирования баз данных

#### Отзывы
- Возможность оставить отзыв только после завершения обучения
- Модальное окно для ввода отзыва
- Отображение отзывов в заявках

### Модуль 3: Оптимизация и анимации

#### Анимации
- Плавные переходы между слайдами
- Анимация появления карточек заявок
- Hover-эффекты на интерактивных элементах
- Микро-анимации кнопок и форм
- Анимация уведомлений

#### Адаптивный дизайн
- Полная поддержка мобильных устройств
- Оптимизация для разрешения 390×844px (iPhone)
- Адаптивная сетка для карточек
- Мобильная навигация

## Структура базы данных

### Таблица `users`
- `id` - первичный ключ
- `login` - уникальный логин
- `password` - пароль (в продакшене должен быть хэширован)
- `full_name` - полное имя
- `phone` - телефон
- `email` - email
- `created_at` - дата создания

### Таблица `courses`
- `id` - первичный ключ
- `name` - название курса
- `description` - описание
- `duration` - продолжительность
- `price` - цена
- `created_at` - дата создания

### Таблица `applications`
- `id` - первичный ключ
- `user_id` - внешний ключ на users.id
- `course_name` - название курса
- `start_date` - желаемая дата начала
- `payment_method` - способ оплаты
- `status` - статус (по умолчанию "Новая")
- `feedback` - отзыв пользователя
- `created_at` - дата создания
- `updated_at` - дата обновления

## API Endpoints

### Аутентификация (`/backend/api/auth.php`)
- `POST` - Регистрация, вход, выход
  - `action: 'register'` - регистрация
  - `action: 'login'` - вход
  - `action: 'logout'` - выход

### Заявки (`/backend/api/applications.php`)
- `GET` - Получить заявки (все для админа, свои для пользователя)
- `POST` - Создать новую заявку
- `PUT` - Обновить статус заявки (только админ)
- `PATCH` - Добавить отзыв к заявке

### Курсы (`/backend/api/courses.php`)
- `GET` - Получить список всех курсов
- `POST` - Создать курс (только админ)

## Учетные данные администратора
